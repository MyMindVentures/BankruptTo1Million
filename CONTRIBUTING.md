# Contributing to Bankrupt to 1 Million

Thank you for helping build the platform behind this mission.

## Before You Start

1. Choose an open GitHub Issue.
2. Comment `I'd like to work on this.`
3. Wait until the issue is assigned.
4. Read [DESIGN_MANIFESTO.md](./DESIGN_MANIFESTO.md).
5. Build only the assigned scope.

Each issue is intended to remain small and focused. Unless explicitly stated otherwise, the expected effort is no more than two hours.

## Contribution Standard

Every contribution must:

- follow the issue acceptance criteria;
- preserve the existing architecture and visual language;
- reuse approved components and design tokens;
- avoid unrelated redesigns or dependencies;
- work across the required responsive breakpoints;
- include accessible labels, focus states and keyboard behavior;
- avoid invented profiles, claims, milestones or social proof;
- document testing and known limitations honestly.

## Branches

Use a focused branch name:

```text
feature/issue-number-short-title
fix/issue-number-short-title
docs/issue-number-short-title
test/issue-number-short-title
```

## Pull Requests

One issue should normally produce one Pull Request.

Include:

- a concise summary;
- `Closes #issue-number`;
- screenshots or a short video for visual changes;
- tests and checks performed;
- known limitations or follow-up work.

## Design Review

A visual contribution is not ready to merge unless it:

- strengthens the mission and documentary story;
- feels human, intentional and credible;
- follows the approved visual system;
- avoids generic AI-generated or SaaS-template aesthetics;
- works on mobile and with keyboard navigation;
- remains within the assigned issue.

The complete standard is defined in [DESIGN_MANIFESTO.md](./DESIGN_MANIFESTO.md).

## Contribution Model

Contributions are voluntary unless an issue explicitly states otherwise. Participation does not automatically create employment, equity, ownership or revenue-share rights. Any different arrangement must be agreed separately in writing.

Meaningful merged contributions receive permanent GitHub credit and may, with permission, be recognized on the Founding Heroes Wall.
